# 🎫 دليل تطبيق العروض الترويجية على الباقات

## 📋 نظرة عامة

هذا الدليل يشرح كيفية تطبيق العروض الترويجية (Promotional Offers) على باقات الاشتراك المختلفة.

---

## 🎯 حقل `applicablePlans` - المفتاح الأساسي

```typescript
interface PromotionalOffer {
    applicablePlans: string[];  // قائمة IDs الباقات المطبقة
}
```

### القواعد:

| قيمة applicablePlans | السلوك | مثال |
|---------------------|--------|------|
| `[]` (فارغ) | يُطبق على **جميع** الباقات | عروض عامة |
| `["subscription_annual"]` | يُطبق على **باقة واحدة** فقط | خصم السنوية فقط |
| `["subscription_6months", "subscription_annual"]` | يُطبق على **باقات محددة** | الباقات طويلة المدى |

---

## 💡 أمثلة عملية

### مثال 1️⃣: عرض على جميع الباقات

```javascript
{
    title: "خصم رمضان الكريم",
    discountType: "percentage",
    discountValue: 20,
    applicablePlans: [],  // ← فارغ = جميع الباقات
    
    startDate: "2025-03-01",
    endDate: "2025-04-01",
    isActive: true
}
```

**التطبيق:**
```
باقة 3 شهور (20$):
  السعر الأصلي: 20$
  الخصم: 20% = 4$
  السعر النهائي: 16$ ✅

باقة 6 شهور (35$):
  السعر الأصلي: 35$
  الخصم: 20% = 7$
  السعر النهائي: 28$ ✅

باقة سنوية (60$):
  السعر الأصلي: 60$
  الخصم: 20% = 12$
  السعر النهائي: 48$ ✅
```

---

### مثال 2️⃣: عرض على الباقة السنوية فقط

```javascript
{
    title: "خصم حصري للاشتراك السنوي",
    discountType: "fixed",
    discountValue: 15,
    applicablePlans: ["subscription_annual"],  // ← السنوية فقط
    
    isActive: true
}
```

**التطبيق:**
```
باقة 3 شهور (20$):
  ❌ لا يُطبق (ليست ضمن applicablePlans)
  السعر النهائي: 20$

باقة 6 شهور (35$):
  ❌ لا يُطبق (ليست ضمن applicablePlans)
  السعر النهائي: 35$

باقة سنوية (60$):
  ✅ يُطبق
  السعر الأصلي: 60$
  الخصم: 15$ (ثابت)
  السعر النهائي: 45$
```

---

### مثال 3️⃣: عرض على باقات متعددة

```javascript
{
    title: "خصم الباقات طويلة المدى",
    discountType: "percentage",
    discountValue: 10,
    applicablePlans: [
        "subscription_6months",
        "subscription_annual"
    ],  // ← 6 شهور والسنوية فقط
    
    minPlayers: 10,  // شرط إضافي
    isActive: true
}
```

**التطبيق (مع 15 لاعب):**
```
باقة 3 شهور (20$):
  ❌ لا يُطبق (ليست ضمن applicablePlans)
  السعر: 20$ × 15 = 300$

باقة 6 شهور (35$):
  ✅ يُطبق (ضمن applicablePlans + عدد اللاعبين ≥ 10)
  السعر الأصلي: 35$ × 15 = 525$
  الخصم: 10% = 52.5$
  السعر النهائي: 472.5$

باقة سنوية (60$):
  ✅ يُطبق
  السعر الأصلي: 60$ × 15 = 900$
  الخصم: 10% = 90$
  السعر النهائي: 810$
```

## 🔄 منطق التطبيق (Flow)

### خطوات التحقق بالترتيب:

```javascript
for (const offer of availableOffers) {
    let applicable = true;
    
    // 1️⃣ تحقق من الباقات المطبقة
    if (offer.applicablePlans && offer.applicablePlans.length > 0) {
        if (!offer.applicablePlans.includes(selectedPackage)) {
            applicable = false;  // ❌ الباقة ليست ضمن القائمة
        }
    }
    // إذا كان فارغ [] → يُطبق على الجميع
    
    // 2️⃣ تحقق من الحد الأدنى للاعبين
    if (offer.minPlayers && selectedCount < offer.minPlayers) {
        applicable = false;  // ❌ عدد اللاعبين أقل من المطلوب
    }
    
    // 3️⃣ تحقق من الحد الأدنى للمبلغ
    const totalAmount = packagePrice * selectedCount;
    if (offer.minAmount && totalAmount < offer.minAmount) {
        applicable = false;  // ❌ المبلغ أقل من المطلوب
    }
    
    // 4️⃣ إذا نجح في كل الفحوصات
    if (applicable) {
        // حساب الخصم واختيار الأفضل
    }
}
```

## 📊 سيناريوهات متقدمة

### سيناريو A: عدة عروض متاحة

```javascript
// العرض الأول
{
    title: "خصم عام",
    discountValue: 10,
    discountType: "percentage",
    applicablePlans: []  // جميع الباقات
}

// العرض الثاني
{
    title: "خصم السنوية المميز",
    discountValue: 25,
    discountType: "percentage",
    applicablePlans: ["subscription_annual"]  // السنوية فقط
}
```

**النتيجة (باقة سنوية 60$):**
- العرض الأول: 10% = 6$ خصم
- العرض الثاني: 25% = 15$ خصم
- **يُطبق الأفضل (العرض الثاني)** ✅
- السعر النهائي: 45$

---

### سيناريو B: عرض + كود شريك

```javascript
// العرض التلقائي
{
    discountValue: 20,
    discountType: "percentage",
    applicablePlans: ["subscription_annual"]
}

// سعر الشريك
{
    partnerCode: "SAFF2024",
    customPricing: {
        yearly: 45  // سعر مخصص للسنوية
    }
}
```

**التطبيق:**
```
1. يُطبق سعر الشريك أولاً:
   60$ → 45$ (سعر الشريك)

2. ثم يُطبق العرض على السعر الجديد:
   45$ → 45$ - (45$ × 20%) = 36$

السعر النهائي: 36$
إجمالي التوفير: 24$ (40%)
```

## 🛠️ كيفية إنشاء عرض مُطبق على باقات محددة

### في واجهة إدارة العروض:

#### 1️⃣ النقر على "إنشاء عرض جديد"

#### 2️⃣ ملء البيانات الأساسية:
```
العنوان: "خصم الباقات طويلة المدى"
الخصم: 15%
التواريخ: من 2025-01-01 إلى 2025-12-31
```

#### 3️⃣ في قسم "الباقات المطبقة":

**الخيارات:**

| الاختيار | النتيجة |
|---------|---------|
| عدم تحديد أي باقة | ✅ يُطبق على **جميع** الباقات |
| تحديد باقة واحدة | ✅ يُطبق على الباقة المحددة **فقط** |
| تحديد عدة باقات | ✅ يُطبق على الباقات المحددة **فقط** |

**مثال:**
```
☐ باقة 3 شهور
☑ باقة 6 شهور
☑ باقة سنوية
```
**النتيجة:** `applicablePlans: ["subscription_6months", "subscription_annual"]`

## ✅ الخلاصة

| العنصر | الوصف |
|--------|-------|
| **applicablePlans: []** | عرض عام على **جميع** الباقات |
| **applicablePlans: [id]** | عرض على **باقة محددة** |
| **applicablePlans: [id1, id2]** | عرض على **باقات محددة** |
| **التطبيق** | تلقائي (إذا لم يكن له code) أو يدوي (مع code) |
| **الأولوية** | الأفضل خصماً يُطبق تلقائياً |
| **مع الشركاء** | سعر الشريك → ثم العرض |

## 🎯 أفضل الممارسات

### ✅ DO (افعل):
- استخدم `[]` للعروض العامة (رمضان، اليوم الوطني)
- حدد باقات محددة للعروض الاستراتيجية (تشجيع الاشتراكات طويلة المدى)
- اجمع بين `applicablePlans` و `minPlayers` لاستهداف دقيق

### ❌ DON'T (لا تفعل):
- لا تنس تحديد الباقات إذا كان العرض خاصاً
- لا تُفعّل عروضاً متضاربة على نفس الباقات
- لا تنسَ تحديث التواريخ
